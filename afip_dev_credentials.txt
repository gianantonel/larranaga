AFIP SDK - Credenciales de DESARROLLO / HOMOLOGACIÓN
======================================================

Fuente: https://docs.afipsdk.com/integracion/api
        https://docs.afipsdk.com/integracion/python

CUIT de prueba: 20-40937847-2
CUIT (numérico, para el SDK): 20409378472

Clave Fiscal:
  No aplica para los Web Services (wsfe, padrón, etc.): en modo `dev` NO
  se requiere certificado ni clave fiscal. El access_token de AFIP SDK
  es lo único necesario para autenticar contra el SDK, y este CUIT de
  prueba funciona contra el entorno de homologación de ARCA sin cert.

  Para las AUTOMATIZACIONES (scraping de portales con clave fiscal),
  AFIP SDK no publica una clave fiscal de prueba genérica: cada
  contribuyente aporta la suya. Para pruebas internas del estudio usar
  las credenciales reales del cliente marcado como "sandbox" en la DB.

Entorno (param `environment`):
  - "dev"  -> homologación (usar con CUIT 20409378472)
  - "prod" -> producción (requiere cert + key del cliente real)

Ejemplo mínimo Python (dev):

    from afip import Afip
    afip = Afip({
        "CUIT": 20409378472,
        "access_token": "<ACCESS_TOKEN del archivo AFIP_SDK_ACCESS_TOKEN/>"
    })
    print(afip.ElectronicBilling.getServerStatus())

Ejemplo REST (dev):

    POST https://app.afipsdk.com/api/v1/afip/auth
    Authorization: Bearer <ACCESS_TOKEN>
    Content-Type: application/json

    {
      "environment": "dev",
      "tax_id": "20409378472",
      "wsid": "wsfe"
    }
